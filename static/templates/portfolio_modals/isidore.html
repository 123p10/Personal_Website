Isidore

December 2019 - Present

Isidore Github

Isidore is a programming language I built from the ground up. I started the Isidore project during the Winter break of 2019. The project is currently in very early development but it supports many modern language constructs and can run incredibly complex programs.

I built my own lexer, AST Tree builder and JIT compiler using LLVM in in C++

I started this project to solve some of the common issues I have experienced in new and old programming languages.

Isidore's current primary goal is to produce readable, maintainable code. The modern programmer values performance and memory efficiency in languages when the priority should be in readibility. As a team member on a project it should not take me more than 3 reads of your function to interpret what and how it performs its operations. Many will solve this issue by commenting inside their function, nearly every line. This just results in more clutter, comments should be used only for incredibly complex operations and for quick summaries of functions(so I don't have to read your code).

I love the features of the new popular languages like Python and Golang(this website is built in it) but certain language constructs can result in messy, confusing code. Python's lack of curly braces for functions can result in unreadable code, as a programmer it is important to easily identify where loops and blocks begin and end. Golang and Python both lack semicolons which I view as another sin of the new set of languages. The semicolon defines the end of a statement, it is not difficult for the compiler to infer the end of statements(isidore used to do this) but the semicolon is not for the compiler it is for the reader. The semicolon acts like the period in sentences, I can understand your sentence without a period, it may look prettier without that period but it becomes easier to understand your message when you explicitly define the end of a statement.

I also love the features of more classic languages like C++(what isidore is written in) and C but I also feel certain "features" hinder it's use. One of the features I hate is when accessing more than one value from a function requires me to pass by reference a variable into the parameters. This is disgusting. Parameters is data passed into a function and for primitive types should not be pass by reference. Any primitive values I desire from the function should be outputted as returns(like in Golang). There are also incredibly confusing and annoying constructs like the vector. The vector is a critically important feature but it uses pretentious, unreadable wording. push_back() is the standard method for appending a value to a list a ridiculous name for such a common feature, could be named append() or insert_end(). There may be important reasons for naming vector the way it is but the vector is simply a dynamic array and could have a much simpler, easier name.

Documentation for the project is currently poor but hopefully in the future on the github link above there will be a features list and plan for isidore.
